# The Agentic Operating Model — Project Brief

## Project Overview

**Title**: The Agentic Operating Model
**Subtitle**: Versioned, AI-Agent-Assisted Work for Code, Operations, Research, and Correspondence
**Duration**: 2–4 hours (workshop/presentation format)
**Target Audience**: PowerShell Developers, DevOps Engineers, System Engineers, Research/Knowledge Workers
**Primary Tool**: GitHub Copilot Agent Mode (VS Code)

## Core Focus

> **This is a presentation about the AGENTIC OPERATING MODEL as a paradigm — a universal pattern for versioned, agent-assisted knowledge work. It is NOT a PowerShell training, and coding is only one instance of the pattern.**

PowerShell is used for demonstrations because the primary audience is familiar with it. The concepts apply to any language and any knowledge-work domain (infrastructure, research, legal, business).

## Core Objectives

1. **Define the Agentic Operating Model**: Explain the universal loop — versioned context → agent action → verification → review — that applies to code, operations, research, and correspondence alike
2. **Establish Context Importance**: Show how Git + memory-bank provides AI with durable understanding
3. **Demonstrate Autonomous Execution**: AI that plans, acts, and verifies independently
4. **Teach Self-Verification**: How automated testing and deterministic artefacts enable AI to validate its own work
5. **Enable Configuration**: How instruction files, custom agents, and skills control AI behavior
6. **Maintain Traceability**: Using Git to track every AI change

## Key Concepts to Convey

### The Paradigm Shift
| Traditional AI | Agentic AI |
|----------------|------------|
| Suggests code | Executes tasks |
| Single-file focus | Repository-wide context |
| Human implements | AI implements, human reviews |
| Human runs tests | AI runs tests and iterates |

### Core Principles
1. **Context is Everything** - Git structure gives AI understanding
2. **Autonomous Execution** - AI acts, not just advises
3. **Self-Verification** - AI tests its own work
4. **Instruction-Driven** - Instruction files and custom agents control behavior
5. **Human Oversight** - Review, approve, rollback
6. **Traceability** - Track all AI changes via Git
